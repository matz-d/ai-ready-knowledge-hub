# Project context（正本ポインター）

## 唯一の正（canonical）

製品の目的、課題設定、ワンライナーなど **プロダクトとして何であるか** の記述は、**リポジトリ直下の [`CLAUDE.md`](CLAUDE.md) のみ** とする。ここに同じ段落を複製しない。

作業を始める前や、プロダクトの境界が絡む判断をするときは **`CLAUDE.md` を読む**。

## `.claude/skills/` との関係

同梱スキル（例: Cloud Run、Gemini API、GCP 認証など）は **手順・API・プラットフォームの作法** を説明する。それらは **`CLAUDE.md` と矛盾してはならない**。

| 種類 | 正の所在 |
|------|-----------|
| 何のためのプロダクトか、誰向けか、何をしないか | **`CLAUDE.md` のみ** |
| gcloud / SDK / 設定コマンドなど | 各ドメインスキル |

指示が衝突したときは、**プロダクト意図は `CLAUDE.md` が優先**。CLI フラグや API の形など **メカニクスは該当スキル** が優先。
