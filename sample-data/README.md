# Sample Data

このディレクトリ配下のサンプルデータは **すべて完全に架空** です。

- 事務所名・会社名・人名・住所・電話番号・銀行口座等 — すべて架空
- 実在の事務所・企業・個人とは一切関係ありません
- 公開資料 (厚労省モデル就業規則、国税庁年末調整資料、日本年金機構ガイド等) は文体・体裁の参考にのみ使用しており、本体テキストはすべて架空のフィクション

## 目的

AI-Ready Knowledge Hub の各エージェント (Curator / Masker / Strategist / Interviewer) の挙動を確認するためのデモ用データ。
社会保険労務士事務所のSME業務を想定して構成している。

## 構成方針

- 全10ファイル (3ペア × 2 + 単体4件)
- 「テンプレ版」と「実案件版」のペアを3組用意し、**テンプレと実案件の違いがエージェントの判定を変える** ことを示す
- ペア構造を全文書に網羅することはしない (詳細は `docs/scope.md` の「サンプルデータ方針」を参照)

## 各ファイルが想定する挙動

### ペア構造

| # | ファイル | 想定される判定挙動 |
|---|---|---|
| 1 | `accounting-office/顧問契約書テンプレ.md` | S1+S3 (テンプレ例外) → Confidential のまま、AI参照版生成 |
| 1 | `accounting-office/顧問契約書_実案件サンプル.txt` | R1 (3要素以上の個人/組織情報) + R2 (顧問料+業務範囲) + R6 (番地レベル住所) → **Masker → Curator 逆feedback で Restricted 格上げ** |
| 2 | `accounting-office/顧客対応メモ_書式.md` | S1+S3 (書式テンプレ) → Confidential のまま |
| 2 | `accounting-office/顧客対応メモ_匿名化.txt` | S2 (placeholder のみ) → 通常マスク後 AI-safe として保持 |
| 3 | `accounting-office/給与計算チェックリスト.md` | 一般情報 → Internal、AI参照可 |
| 3 | `accounting-office/給与計算_例外対応メモ.txt` | 一般知識として使えるが個別情報不足 → **Strategist が不足知識・確認質問を出す** |

### 単体

| ファイル | 想定挙動 |
|---|---|
| `accounting-office/就業規則テンプレート.md` | 汎用テンプレ。Internal、AI参照可 |
| `accounting-office/年末調整_案内文.txt` | 顧客向け案内文。Internal、AI参照可 |
| `accounting-office/料金表_2026.csv` | 現行料金表。Internal、鮮度=現行 |
| `accounting-office/古い料金表_2023.csv` | 旧版料金表。Curator 鮮度判定で「旧版候補」フラグが立つ想定 |

## 注意

- 個人情報の演出度合い (氏名・住所・金額・日付の具体性) は Masker の挙動を見せるための調整であり、実在の情報ではありません
- 電話番号は `XXXX` 部分などを架空値にしており、実存しません
- このディレクトリのデータは Git にコミットして問題ありません (本物の機密情報は含まれない)
